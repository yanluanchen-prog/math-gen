# WEB_GAME_WORKFLOW.md（下次你说“我要开发网页了”时，我会按这个流程带你走）

## 0) 先定方向（10 分钟）
1. 填 `PRD_ONEPAGER.md`
2. 明确：这版“必须做”只有 3 个核心功能
3. 明确：这版“不做清单”（防止越做越大）

## 1) 搭骨架（30-60 分钟）
- 先把页面结构/路由（screen 切换）搭出来
- 先做一个“能玩一题”的最小闭环

## 2) 版本策略（从第一天开始）
- 母版：持续开发文件
- 锁定版：每次里程碑复制一份（带时间戳）
- `VERSIONS.md` 记录：文件名 + SHA256 + 备注

## 3) 每次迭代只做一件事
- 主题例子：只改提示、只改音效、只改地图、只改存档
- 做完立刻跑 `TEST_CHECKLIST.md`

## 4) 常见坑位（这次已经踩过）
- 自动播放限制（音效/BGM/语音）→ 必须“首次用户点击后”再初始化音频
- 单文件 HTML 很容易改崩 → 用锁定版 + 小步提交
- inline onclick / 作用域 / 语法报错 → 任何一个报错会让一堆按钮失灵
- 预览环境（如飞书预览）可能限制 JS → 用真实浏览器/本地 http server 验证

## 5) 我会怎么问你（下次开工的问诊）
- 用户是谁？要解决什么？成功标准是什么？
- 这版只做哪 3 个功能？哪些明确不做？
- 你希望在哪发布（GitHub Pages/本地/飞书）？
- 需要离线吗？需要存档吗？

> 你只要说“我要开发网页了”，我会先让你填一页纸，再带你走最小闭环，然后再扩展。
